FlyCast API Overview

FlyCast is a RESTful web API that predicts the probability of a flight delay based on airline, schedule, and route information.
API Host: https://flycast-api.onrender.com

Features

  • Predicts flight delay probability (0–1)
  • Instant binary delay prediction based on an optimal threshold (default: 0.80)
  • Plug-and-play endpoint, no authentication required
  • Supports all US airlines/major US airports (expandable)

Quick Start

Get started with FlyCast API in seconds. Simply send a POST request to our prediction endpoint with flight details.

Base URL

All API requests should be made to: https://flycast-api.onrender.com

Basic Usage:
curl -X POST https://flycast-api.onrender.com/predict \
  -H "Content-Type: application/json" \
  -d '{"AIRLINE": "Delta Air Lines Inc.", "DAY_OF_WEEK": 3, "DEP_HOUR": 10, "MONTH": 9, "dep": "LAX", "arr": "JFK"}'

API Reference

Complete reference for the FlyCast API endpoints and parameters.

POST /predict

Predict the probability of a flight delay given standard flight details.

  • URL: https://flycast-api.onrender.com/predict
  • Method: POST
  • Content-Type: application/json

Request Parameters

Parameter Type Required Description
AIRLINE string Yes Airline name or IATA code
DAY_OF_WEEK int Yes Day of week (1=Monday, ..., 7=Sunday)
DEP_HOUR int Yes Scheduled departure hour (0-23)
MONTH int Yes Month number (1-12)
DELAY_DUE_WEATHER int No 1 if prior weather delay, else 0 (default: 0)
DELAY_DUE_NAS int No 1 if prior NAS (airspace) delay, else 0 (default: 0)
DELAY_DUE_CARRIER int No 1 if prior carrier delay, else 0 (default: 0)
dep string Yes IATA code of departure airport (e.g., "LAX")
arr string Yes IATA code of arrival airport (e.g., "JFK")

Response Format

Success Response:
{
  "delay_probability": 0.4483,
  "prediction": 0
}
  • delay_probability: Probability (0–1) that the flight will be delayed
  • prediction: 1 = Delayed (if probability >= 0.8), 0 = On time
Error Response:
{
  "error": "Missing field: AIRLINE"
}

Ready-to-use code examples for integrating FlyCast API into your applications.

curl -X POST https://flycast-api.onrender.com/predict \
  -H "Content-Type: application/json" \

    "DELAY_DUE_WEATHER": 0,
    "DELAY_DUE_NAS": 0,
    "DELAY_DUE_CARRIER": 0,
    "dep": "LAX",
    "arr": "JFK"
  }'

JavaScript Example


								
const flightData = {
  AIRLINE: "Delta Air Lines Inc.",
  DAY_OF_WEEK: 3,
  DEP_HOUR: 14,
  MONTH: 9,
  DELAY_DUE_WEATHER: 0,
  DELAY_DUE_NAS: 0,
  DELAY_DUE_CARRIER: 0,
  dep: "LAX",
  arr: "JFK"
};

fetch('https://flycast-api.onrender.com/predict', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify(flightData)
})
.then(response => response.json())
.then(data => {
  console.log('Delay probability:', data.delay_probability);
  console.log('Prediction:', data.prediction ? 'Delayed' : 'On Time');
})
.catch(error => console.error('Error:', error));

Python Example

import requests
import json

# Flight data
flight_data = {
    "AIRLINE": "American Airlines Inc.",
    "DAY_OF_WEEK": 5,
    "DEP_HOUR": 8,
    "MONTH": 12,
    "DELAY_DUE_WEATHER": 0,
    "DELAY_DUE_NAS": 0,
    "DELAY_DUE_CARRIER": 0,
    "dep": "DFW",
    "arr": "LAX"
}

# Make API request
response = requests.post(
    'https://flycast-api.onrender.com/predict',
    headers={'Content-Type': 'application/json'},
    data=json.dumps(flight_data)
)

if response.status_code == 200:
    result = response.json()
    print(f"Delay probability: {result['delay_probability']:.4f}")
    print(f"Prediction: {'Delayed' if result['prediction'] else 'On Time'}")
else:
    print(f"Error: {response.json()}")

Developer Notes

Technical details and implementation guidance for integrating FlyCast API.

How It Works

  1. Receives flight details via POST to /predict
  2. Preprocesses categorical info (airline, airports)
  3. Looks up airport-to-airport distance
  4. Feeds formatted input to the ML model
  5. Returns probability and binary prediction

Machine Learning Model

FlyCast uses a trained machine learning model that considers multiple factors including historical delay patterns, weather conditions, airline performance, and route characteristics to predict flight delays.

Project Structure

pred_model/
│
├── flight_prediction_api.py      # API source code (Flask)
├── airport_distances.pkl         # Precomputed airport distance matrix
├── flight_delay_model.pkl        # Serialized ML model
├── model_columns.pkl             # Feature columns for the model
└── requirements.txt              # Python dependencies

Best Practices

Important Notes

  • Send only IATA codes for dep and arr: e.g., "JFK", "LAX"
  • All categorical fields are internally encoded
  • delay_probability is a float; prediction uses an 0.80 threshold
  • You may omit DELAY_DUE_WEATHER, DELAY_DUE_NAS, and DELAY_DUE_CARRIER—they default to 0
  • Distance between airports is automatically computed; do not include it in requests
  • No authentication is currently required
